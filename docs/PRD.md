# Delicious 产品需求文档 (PRD)

> 版本: v2 (2026-06-30 修订)
> 状态: V1.0 设计冻结，进入开发

## 1. 产品定位

一款 AI 驱动的"美食日记 + 厨房助手" App。用户拍一张菜的照片，AI 自动识别菜名、生成材料清单、估算成本、写出制作步骤。记录沉淀为个人菜单，可随时复用、生成购物清单、查看烹饪开销。

**一句话**：拍照即所得的美食日记，让"我做过/我想再做"的菜不再丢失。

## 2. 目标用户与场景

| 用户画像 | 高频场景 | 我们解决的痛点 |
|---|---|---|
| 周末家庭主厨 | 周日做了一桌菜，想留个记录 | 拍照即记录，不用手动写菜谱 |
| 想学做饭的新手 | 看到外卖里的菜想自己做 | AI 给出可执行步骤和购物清单 |
| 精打细算的家庭烹饪者 | 月底想看伙食花了多少钱 | 每道菜估算成本，月度汇总 |
| 健身/饮食控制人群 | 复刻历史菜单 | 一键"再做一次"，按人数缩放 |

## 3. 核心价值主张

1. **拍照即所得** — 不输入文字、不选标签，一张图换一份完整食谱。
2. **AI 是辅助、用户是主** — AI 结果默认就保存，但任何字段都能就地编辑。
3. **从"记录"到"行动"** — 记录的菜可以一键加入购物清单、一键再做一次。
4. **看得见的成本** — 每道菜都有钱数，每周/月都有总账。

## 4. 用户旅程 (Golden Path)

### 4.1 首次启动
```
打开 → 登录/注册 → 3 屏引导卡片 → 立即拍一张 → AI 识别 → 保存第一条 → 首页有内容
```
目标：从冷启动到第一条记录 < 60 秒。

### 4.2 日常记录（核心循环）
```
首页 → 中间凸起按钮 → 拍照/选图 (选完立即开始识别)
  → 识别中 (三段式进度文案)
  → 结果页 (可就地编辑任意字段)
  → 保存 → 回首页
```
目标：拍照启动到保存完成 ≤ 4 次点击、≤ 30 秒。

### 4.3 复用
```
首页/搜索 → 点开一道菜 → "再做一次" → 调整人数 → 复制到今天
                       → "加入购物清单" → 自动合并食材
```

### 4.4 失败兜底
识别失败 / 用户不满意 → 三个出口：**换一张图** / **手动填写** / **反馈"识别不准"**（写入 ai_feedback，供 prompt 迭代）

### 4.5 出门买菜（V1.1 引入）
```
购物清单 → 顶部「🧭 看看去哪买」
  → 询问定位授权（已授权则跳过）
  → 3 Tab：附近超市 / 配送到家 / 外卖买菜
  → AI 一句话推荐策略：「永辉买 7 项 + 叮咚补紫苏，约 ¥40 最省」
  → 点店铺：跳第三方导航/App，或回退到网页
```
设计原则：信息密度做满（一眼看完三个 Tab 就知道选哪家），让用户跳出之前决策已经完成。

## 5. 信息架构

底部 Tab，三个一级入口：

| Tab | 内容 | 默认页 |
|---|---|---|
| 日记 | 时间线 + 搜索 + 分类筛选 | 启动默认页 |
| 拍照 | 中间凸起按钮，唤起识别流程 | — |
| 我的 | 用户信息、本周/月成本面板、购物清单、个人价格表、设置、退出登录 | — |

**重要改动**：原 `home_screen` 右上角的"退出登录"必须迁到「我的」→ 设置；目前误点风险极高。

## 6. 功能列表（按版本）

### V1.0 — MVP+ (本期开发目标)

#### F1 AI 识别与就地编辑（P0）
- 选图后**立即触发识别**，不要"开始识别"按钮
- 结果页和编辑页**合并为一页**：默认漂亮排版，每个字段点击进入行内编辑
- 顶栏只有一个动作："保存到日记"
- 材料/步骤支持**增/删/改**：
  - 每条右侧编辑、删除图标
  - 列表末尾"+ 添加一项"
  - 步骤支持长按拖拽排序
- Tips 同样可增删

#### F2 识别中进度提示（P0）
三段式假进度（无需后端流式）：
- 0–2s：「识别食物中…」🍅
- 2–5s：「估算成本中…」💰
- 5s+：「生成制作步骤中…」📝

#### F3 识别失败的三个出口（P0）
错误页提供：换一张图 / 手动填写 / 反馈识别不准（多选 + 备注，POST `/ai/feedback`）。

#### F4 底部 Tab 导航 + 我的页（P1）
- 三个 Tab；中间为拍照凸起按钮
- 「我的」首屏：头像 + 昵称 + 本周成本卡片 + 入口列表（购物清单、价格表、设置、退出）

#### F5 首次引导 + 空状态强化（P1）
- 首次登录后 3 屏卡片（拍照即可记录 / AI 自动算成本 / 一键再做和买菜）
- 末屏 CTA："立即拍第一张"，直接唤起相机

#### F6 购物清单（P1，本期最重要的留存抓手）
- 食谱详情页：「加入购物清单」按钮
- 我的 → 购物清单页：
  - 已勾选 / 未勾选分组
  - 同名 + 同单位的食材自动合并数量
  - 长按可调整数量
  - "全部清空已勾选" 一键键
  - "分享给家人"（文本形式分享）

#### F7 再做一次（P1）
- 食谱详情页 FAB「再做一次」
- 弹窗确认人数（默认沿用原食谱）
- 按比例缩放材料数量和成本
- 复制为今天的新记录（cooked_at = today），原记录不动

#### F8 按食材搜索（P1）
- 首页搜索框增加 tab 切换：菜名 / 食材
- 后端 `GET /foods` 增加 `ingredient` 参数（详见 API.md）

#### F9 个人价格记忆（P1）
- 用户修改材料价格 → 写入 `user_ingredient_prices`
- 下次 AI 识别命中相同食材时，**用用户的最新价格覆盖** AI 估算
- 我的 → 我的价格表：可查看 + 手动调整 + 删除
- 列表展示：食材名 / 单价 / 单位 / 上次更新

#### F10 成本面板（P1）
- 我的页首屏卡片：本周烹饪支出 + 平均每餐成本
- 点开 → 月度详情页：
  - 月度总成本柱图（按周分组）
  - 最贵 Top 3 / 最便宜 Top 3
  - 同分类支出占比饼图

#### F11 复购检测（P1）
- 保存时若同月内有同名菜或材料重合 ≥ 85%，后端返回 200 + duplicate_of
- 客户端弹窗三选项：更新原记录 / 新建一条 / 取消
- 详见 [API.md POST /foods] 与 [ARCHITECTURE §4.3]

### V1.1 — 留存与变现（V1.0 上线后 2~4 周）

#### F12 购物建议「看看去哪买」（P0，V1.1 头号优先级）

让用户从"列清单"走到"真的买到"。AI 的核心价值不是"番茄哪里有"，而是"这一整张清单去哪儿买最高效"。

- **入口**：购物清单顶部新增按钮「🧭 看看去哪买」（清单非空时显示）
- **位置授权**：首次点击弹系统授权；拒绝则回退到"按城市选择"
- **三个 Tab**：
  - **附近超市**：高德 POI 按距离排序，标注覆盖度（X/N 项）和缺哪几样
  - **配送到家**：盒马 / 叮咚 / 朴朴等同城配送 App
  - **外卖买菜**：美团买菜 / 京东到家
- **AI 推荐策略**：基于清单 + 用户位置 + 可达店铺，生成一句话最优组合
- **跳转**：点店铺 → 高德/百度地图导航；点 App → deeplink，失败回退到 web fallback
- **AI 该做与不该做**：
  - ✅ 食材语义归类（番茄=西红柿）、店铺类型对清单的覆盖度判断、一句话推荐文案
  - ❌ 找门店（用高德 API）、抓商品价格（V1.1 不做，反爬严重）、做比价
- **隐私**：坐标精度 100 米脱敏存储，仅用于本次推荐；用户可一键关闭定位
- 详见 [ARCHITECTURE §5 购物建议系统] 与 [API.md POST /shopping/buy-suggestions]

#### F13 周报推送（P1）
每周日 20:00 push「这周做了 X 道菜，估算省下 ¥Y 外卖费」

#### F14 分享卡片差异化（P2）
菜谱长图 / 纯文本食谱 / 购物清单三种格式，每张带 App 水印

#### F15 离线草稿（P2）
识别后没保存就退出，下次进入提示「上次有一份草稿未保存」

#### F16 token 持久化恢复（P1，技术补丁）
启动时尝试用 secure storage 里的 token 直接进入登录态，避免每次启动都要重登

## 7. 关键交互细则

### 7.1 就地编辑约定
- 菜名：点击 → 输入框 + 软键盘 + 确认/取消按钮
- 单条材料：点击 → bottom sheet，包含 名称 / 数量 / 单位 / 单价（自动同步个人价格表）
- 单条步骤：点击 → bottom sheet，描述（多行文本）+ 耗时（数字）
- 删除：右侧 X 图标 + 一次确认；删除后顶部出现「撤销」snackbar

### 7.2 购物清单合并算法
- key = (规范化名称, 单位)；同 key → 数量相加
- 不同单位（如「番茄 2 个」vs「番茄酱 1 瓶」）保留为两条
- 名称规范化：去空格、繁简归一、忽略大小写
- 用户手动添加的条目优先级高于自动加入的

### 7.3 个人价格覆盖
- AI 返回的 `estimated_price` 为 fallback
- 后端在 `/ai/recognize` 响应前查 `user_ingredient_prices`：命中则用个人价格替换，并在响应里标记 `price_source: "user"`
- 客户端 UI 上对用户校准过的价格显示一个小角标

### 7.4 退出登录确认
两步确认（弹窗 → 确认按钮），避免误退导致草稿丢失。

### 7.5 购物建议交互细则（V1.1）
- **位置授权时机**：仅在用户点了「看看去哪买」之后请求，不在 App 启动时弹（避免劝退）
- **店铺排序**：附近超市按 `score = 0.6 * 覆盖率 + 0.4 * (1 - 距离/最大距离)`，覆盖优先于距离
- **缺货降级**：超市覆盖不到 60% 时，AI 文案会主动建议"补 X 项可走叮咚"，把用户引到第二个 Tab
- **deeplink 兜底**：每条线上链接同时附带 web URL，跳第三方 App 失败时打开浏览器
- **空数据兜底**：当前定位附近 1 公里无 POI → 显示"换个地方试试"+ 手动输入城市的入口
- **不打扰**：用户已经选过店铺一次，30 分钟内再次点开默认折叠 AI 推荐文案，直接展示上次选择置顶

## 8. 度量与北极星

**北极星**：周活用户人均记录数 ≥ 3

| 指标 | 当前 | V1.0 目标 |
|---|---|---|
| 拍照 → 保存漏斗完成率 | 未度量 | ≥ 70% |
| 编辑率（用户改了 AI 至少一个字段） | 未度量 | ≥ 30% |
| AI 识别 P95 耗时 | 未度量 | < 8s |
| AI 识别准确率（用户未做大范围修改） | 未度量 | > 85% |
| 次日留存 | 未度量 | ≥ 40% |
| 购物清单功能 7 日渗透率 | N/A | ≥ 25% |
| 再做一次月活渗透率 | N/A | ≥ 15% |

### V1.1 新增指标

| 指标 | V1.1 目标 |
|---|---|
| 购物清单用户中点击"看看去哪买"占比 | ≥ 40% |

| 购物建议 → 实际再次打开 App（7 日内回访） | ≥ 60% |

## 10. 非目标

**V1.0 不做**
- 多人协作 / 家庭共享菜单
- 视频步骤
- 智能体对话式修改（"把这道菜改成低盐版"）
- 自建食材库 / 厨具库
- 营养成分

**V1.1 购物建议明确不做**
- 抓取电商页面比价（反爬严重、数据不稳定，会被骂死）
- 用户评价聚合
- App 内承接交易（没那么多资源，跳第三方就行）
- 海外用户（高德覆盖国内，海外用 Google Places 等 V2 再说）

这些放在 V2 评估，不为它们让当期版本加复杂度。

## 11. 风险与依赖

| 风险 | 应对 |
|---|---|
| OpenAI 价格波动 / 限流 | Redis 缓存 + 失败兜底为手动模式 |
| 用户上传非食物图 | Prompt 增加"非食物则返回 unrecognized"指令，前端给出友好提示 |
| 个人价格表脏数据 | 单条价格如偏离市场均价 ±5× 则不参与覆盖，仅展示 |
| 识别准确率不达标 | ai_feedback 表收集 bad case，每周离线分析 prompt 调整 |
| **(V1.1)** 高德 POI API 5000 次/日额度 | 4 层缓存（食材别名 30 天 / POI 6h / 覆盖度 24h / 文案 4h），目标命中率 ≥ 90% |
| **(V1.1)** 用户拒绝定位授权 | 回退到"手动选城市 + 行政区"，仍能给出 POI（精度降低） |
| **(V1.1)** deeplink 不稳定（盒马/美团版本差异） | 每条跳转附带 web URL；记录跳转成功率，季度审视 |
| **(V1.1)** 跳出第三方 App 后不回流 | 入口处把决策辅助做到位（覆盖度 + 一句话推荐），让用户跳出前已决策完成 |
| **(V1.1)** 用户隐私（定位敏感） | 坐标精度脱敏到 100 米存储；仅在请求购物建议时使用；用户可一键关闭 |
